# RBK-002 Critical Vulnerability Response

## Purpose
Coordinate response to critical vulnerabilities affecting cloud or hybrid workloads.

## Steps
1. Validate vulnerability source and affected assets.
2. Assess exposure and prioritize based on criticality.
3. Open remediation ticket and notify asset owner.
4. Apply mitigation or patch within SLA.
5. Verify remediation and update evidence.

## Evidence
- Advisory and CVE references
- Asset list and remediation status
- Verification logs
